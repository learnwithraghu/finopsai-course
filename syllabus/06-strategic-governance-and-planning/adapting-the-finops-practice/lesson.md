# Adapting the FinOps Practice

## Purpose
Evolve the FinOps operating model to sustainably support AI workloads and partner teams.

## Learning Objectives
- Assess current FinOps org design (central vs. federated) and adjust roles for AI specialization
- Prioritize tooling and automation investments that unlock AI visibility at scale
- Define success metrics for the evolved practice (cycle time, adoption, savings)

## Discussion Topics
- Operating model options: hub-and-spoke, embedded FinOps, community of practice
- Tooling backlog (GPU observability, ML platform integrations, automated policies)
- Outcome measurement and continuous improvement

## Practical Activities
- Run a capability gap analysis specifically for AI use cases
- Create a roadmap covering people, process, and technology initiatives
- Set quarterly OKRs for the FinOps practice to track adoption and value

## Artifacts & Metrics
- Capability gap matrix
- Practice roadmap
- OKR set
